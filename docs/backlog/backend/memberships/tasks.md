# Memberships

> Feature: backend

## Revisit Project Invite Flow

**Status:** `done`
**Description:** O fluxo atual de invite de membros adiciona o usuário diretamente como membro ativo do projeto, sem nenhuma etapa de aceite. Decidir entre duas abordagens: (1) manter inserção direta, simplificando o fluxo; (2) implementar fluxo de aceite com status pendente, notificação ao convidado e ação de aceitar/recusar o convite.
**User Story:** As a project owner, I want to define whether invited members join directly or must accept an invite, so that the membership flow matches the expected user experience.
